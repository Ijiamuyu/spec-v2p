### 3.3 分项解读

#### LUT：~66%（合理估算，范围 56%~81%）

评估处于**可行范围**，合理估算 ~66% 低于 75% 安全线，但上限 ~81% 表明需要实测验证以缩小不确定区间。若所有模块同时取高估值则触及阈值，正常预期在 60%~70% 之间。

**主要来源**（按占比排序）：

| 子系统 | 合理估算 | 占比 |
|--------|---------|------|
| AP（四核 RV64 SMP） | ~55K | ~27% |
| NPU（Coral） | ~30K | ~15% |
| LSIO（含 Ethernet / DMA / SD） | ~22K | ~11% |
| 系统基础设施（含 AXI 互联） | ~14K | ~7% |
| FSI（RV32 + 安全外设） | ~14K | ~7% |

**最大不确定性来源**：
- **VexRiscv-SMP RV64 ×4**：单核 LUT 区间 7K~10K，四核 + 缓存一致性合计 **39K~58K**，是最大的单一变量。建议立即做早期综合获取实测数据。
- **Coral NPU**：标量+向量+矩阵 MAC 三合一，实测可能在 30K~50K 区间。
- **AXI 互联**：6 主多从架构下 4K 低估，合理区间 **8K~12K**。

#### BRAM：~64%

片上存储器为主要消耗：

| 项目 | BRAM36 | 占比 |
|------|--------|------|
| SYSTEM SRAM 672KB | ~150 | ~34% |
| FSI 私有 SRAM 128KB | ~30 | ~7% |
| NPU ITCM/DTCM 与内部缓冲 | ~48 | ~11% |
| AP 缓存 / PLIC / CLINT | ~16 | ~4% |
| LSIO 外设缓冲 | ~8 | ~2% |
| BootROM 32KB | ~8 | ~2% |
| AXI Interconnect FIFO | ~8 | ~2% |
| DDR3 MIG / 余量 | ~18 | ~4% |
| **合计** | **~286** | **~64%** |

余量 ~159 个 BRAM36（约 36%），如需可进一步扩展。

#### DSP：~19%

DSP 主要被 Coral NPU 矩阵 MAC 脉动阵列消耗（~130 个），其余子系统合计 ~29 个。余量 ~681 个，**非常充裕**。

## 4. 评估结论

| 维度 | 评估 | 说明 |
|------|------|------|
| **资源充足性** | ✅ **可行** | LUT ~66%（范围 56%~81%），BRAM ~64%，DSP ~19%，合理估算低于 75% 安全线 |
| **时序收敛** | ⚠️ 需关注 | LUT 若超 70% 则布局布线压力上升，建议先做 VexRiscv-SMP 综合验证 |
| **瓶颈资源** | **BRAM** | 片上 SRAM 占用 ~53% BRAM，是后续扩展的主要约束 |
| **扩展余量** | 充裕 | GTX / PCIe / CMT / DSP 均有大量余量 |

### 建议

1. **立即执行**：生成 VexRiscv-SMP 四核 RTL，在 Vivado 中做早期综合，修正 AP 子系统的 LUT 数据
2. **并行推进**：拉取 Coral NPU RTL，在 Vivado 中做综合测试工程，获取真实 LUT/DSP/BRAM 数据
3. **应急方案**：若实测 LUT 突破 75%，优先裁剪 Ethernet GMAC 或缩减 SYSTEM SRAM
