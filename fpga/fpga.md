### 3.3 分项解读

#### LUT：~65%（中位数估算，范围 120K~150K，占用率 59%~74%）

评估处于**可行范围**，中位数 ~65% 低于 75% 安全线，上限 ~74% 仍在安全线内，时序收敛压力较小。

**主要来源**（按占比排序）：

| 子系统 | 中位数估算 | 占比 |
|--------|-----------|------|
| AP（四核 RV64 SMP + Debug） | ~48K | ~36% |
| NPU（Coral） | ~36K | ~27% |
| LSIO（含 Ethernet / DMA / SD / PWM） | ~21K | ~16% |
| FSI（RV32 + 安全外设 + PMU） | ~14K | ~11% |
| 系统基础设施（含 AXI 互联） | ~13K | ~10% |

**最大不确定性来源**：
- **VexRiscv-SMP RV64 ×4**：单核 LUT 区间 9K~11K（含 MMU/Sv39/缓存），四核合计 **36K~44K**，加上缓存一致性/PLIC/CLINT/Debug 约 6K~8K，总计 **42K~52K**，中位数约 48K。
- **Coral NPU**：标量+向量+矩阵 MAC 三合一，控制逻辑开销使实测 LUT 比算术估算高 15%~25%，中位数约 **36K**。
- **AXI 互联**：6 主多从架构，含 CDC 异步 FIFO，合理区间 **8K~10K**，中位数约 8K。

#### BRAM：~63%

片上存储器为主要消耗：

| 项目 | BRAM36 | 占比 |
|------|--------|------|
| SYSTEM SRAM 672KB | ~150 | ~34% |
| FSI 私有 SRAM 128KB | ~30 | ~7% |
| NPU ITCM/DTCM 与内部缓冲 | ~48 | ~11% |
| AP 缓存 / PLIC / CLINT / Debug | ~16 | ~4% |
| LSIO 外设缓冲 | ~8 | ~2% |
| BootROM 28KB | ~8 | ~2% |
| AXI Interconnect FIFO | ~8 | ~2% |
| DDR3 MIG / 余量 | ~13 | ~3% |
| **合计** | **~281** | **~63%** |

余量 ~164 个 BRAM36（约 37%），如需可进一步扩展。

#### DSP：~19%

DSP 主要被 Coral NPU 矩阵 MAC 脉动阵列消耗（~130 个），其余子系统合计 ~29 个。余量 ~681 个，**非常充裕**。

## 4. 评估结论

| 维度 | 评估 | 说明 |
|------|------|------|
| **资源充足性** | ✅ **可行** | LUT ~65%（范围 59%~74%），BRAM ~63%，DSP ~19%，中位数估算低于 75% 安全线，上限仍在安全线内 |
| **时序收敛** | ✅ **压力较小** | LUT 中位数 ~65% 占用下，-2 速度等级跑 **100MHz 系统时钟无压力**；若 DDR3 跑 400MHz（800Mbps）、NPU 跑 150MHz，需提前做物理约束和分区布局 |
| **瓶颈资源** | **BRAM** | 片上 SRAM 占用 ~53% BRAM，是后续扩展的主要约束，建议非实时数据优先搬到 DDR3 |
| **扩展余量** | 充裕 | GTX / PCIe / CMT / DSP 均有大量余量 |

### 建议

1. **验证优先级**：建议先单独综合 **VexRiscv-SMP 四核 + AXI 互联**，这两项占 LUT 总量近 50%，拿到实测数据后即可锁定整体资源水位
2. **并行推进**：拉取 Coral NPU RTL，在 Vivado 中做综合测试工程，获取真实 LUT/DSP/BRAM 数据
3. **BRAM 规划**：当前 63% 占用中片上 SRAM 占绝大部分，后续若要加缓存、扩 NPU 缓冲，优先考虑把非实时数据搬到 DDR3
