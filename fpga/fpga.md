### 3.3 分项解读

#### LUT：~75%

评估处于**可行范围**。

**最大不确定性来源**：VexRiscv-SMP RV64 ×4 的 LUT 面积（单核 ~8.5K）、Ethernet GMAC（~8K）、AXI Interconnect（~10K），建议通过单核综合实测验证。

#### BRAM：~74%

片上存储器为主要消耗：

| 项目 | BRAM36 | 占比 |
|------|--------|------|
| SYSTEM SRAM 704KB | ~157 | ~35% |
| NPU 私有 SRAM 128KB | ~30 | ~7% |
| FSI 私有 SRAM 128KB | ~30 | ~7% |
| Gemmini Scratchpad 40×36Kb | 40 | ~9% |
| AXI Interconnect FIFO | ~16 | ~4% |
| 各外设缓冲 | ~16 | ~4% |
| DDR3 MIG / 其他 | ~8 | ~2% |
| **合计** | **~329** | **~74%** |

余量 ~116 个 BRAM36（约 26%），如需可进一步扩展。

#### DSP：~17%

DSP 主要被 Gemmini 8×8 脉动阵列消耗（~128 个），余量 ~700 个，**非常充裕**。

## 4. 评估结论

| 维度 | 评估 | 说明 |
|------|------|------|
| **资源充足性** | ✅ **可行** | LUT ~75%，BRAM ~74%，DSP ~17%，均在安全线内 |
| **时序收敛** | ⚠️ 偏紧可能 | 若布线复杂度高（多主多从 AXI + 缓存一致性），需关注时序 |
| **瓶颈资源** | **BRAM** | 片上 SRAM 占用 ~64% BRAM，是后续扩展的主要约束 |
| **扩展余量** | 充裕 | GTX / PCIe / CMT / DSP 均有大量余量 |

### 建议

1. **先做单核 VexRiscv-SMP RV64 综合**：获得真实 LUT 数据后修正整体评估
2. **SYSTEM SRAM 按需起步**：若 BRAM 紧张，可将 704KB 裁剪至 256KB（省 ~100 BRAM36）
3. **Ethernet GMAC 可延后**：若时序紧张，QEMU 阶段已验证网络功能，FPGA 阶段可暂不包含
